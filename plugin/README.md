# AI Founder Router

Your AI co-founder + 34 specialist agents. Describe any task and get the right team on it immediately.

## Commands

| Command | Use When |
|---|---|
| `/founder "task"` | Universal — any task, any domain. Your co-founder figures out the rest. |
| `/engineering "task"` | You know it's an engineering task |
| `/product "task"` | You know it's a product/design/strategy task |
| `/business "task"` | You know it's business/ops/data/compliance |

## Examples

```
/founder "build a RAG-powered document Q&A feature"
→ Routes to: ai-architect + ai-engineer + ai-backend-engineer (parallel)

/founder "we need to be GDPR compliant before launch"
→ Routes to: legal-compliance-agent + responsible-ai-engineer (parallel)

/founder "optimize our LLM costs, we're spending $8k/month"
→ Routes to: ai-finance-cost-optimization + prompt-engineer

/engineering "design the database schema for conversation history"
→ Routes to: ai-database-engineer

/product "write a PRD for AI-powered search"
→ Routes to: ai-product-manager

/business "set up a security review for our AI API"
→ Routes to: ai-security-risk
```

## The `founder` Agent

For complex projects needing strategic judgment, use the `founder` agent directly. It thinks like a co-founder — first principles, business context, bias for action — then delegates to the right specialists.

```
Use the founder agent to plan our go-to-market for the next 90 days.
```
